About This Website

Purpose, context, and the story behind the BlueBush project

⚠️
This is a 100% Fictitious Website

BlueBush does not exist as a real business. No products listed on this site are real. No transactions can be made. All addresses, phone numbers, and contact details are entirely fabricated. This website was created solely for educational purposes as part of a TAFE assignment.

What is BlueBush?

BlueBush is a synthetic, fictitious brand — a Perth-based sustainable homewares business — created entirely as a design and AI training exercise. It does not operate as a real store, does not sell products, and has no physical or commercial presence.

The brand was designed to be realistic, believable, and consistent — acting as a "world class" synthetic dataset and web presence for an AI customer service agent training exercise.

Assignment Purpose

This website was generated as part of an assignment for a TAFE AI Skillset Course. The project's primary objective was to:

  • Design a realistic product catalog and AI training dataset for a synthetic e-commerce brand
  • Demonstrate the use of AI tools to generate structured data (product JSON), marketing copy, and training datasets
  • Explore AI customer service agent architecture, including RAG (Retrieval-Augmented Generation) data schema design
  • Build a functional web presence to house and demonstrate the AI training data in context
  • Document risk safeguards against common AI failure modes (greenwashing, phantom stock, etc.)

The BlueBush Project Roadmap

The project followed a structured multi-phase roadmap:

Phase 1
Architecture & Standards
Established brand identity (BlueBush, Perth-based), defined the "Gold Standard" JSON schema separating dynamic data (price, stock) from static data (dimensions, specs), and implemented Content Triage.
Phase 2
Data Engineering
Generated 70+ product entries across Textiles, Hard Goods, Consumables, and Wellness categories — each with full metadata including GSM weights, ingredient lists, care instructions, and RAG resources.
Phase 3
Operational Safeguards
FAQ Gap Analysis across 7 categories, Risk Assessment documentation, and explicit mapping of "Competitor Failures" (e.g., phantom stock) to "BlueBush Solutions" (real-time buffer logic).
Phase 4
Training Assets
Curated 50-item interaction dataset (Q&A pairs), System Prompts based on ICE-AUS and UWA Corpus research (Neutral Professional Australian English, no caricatures).
Phase 5
Strategic Report
Drafted sections covering RAG Architecture (the "Frozen in Time" Paradox), Strategic Risk Mitigation (preventing greenwashing and subscription traps), and Data Limitations.
Phase 6
Web Presence
This website — a fully functional static site demonstrating the product catalog, dynamic product pages, FAQ, chatbot UI, and brand identity consistent with the style guide.

Key Design Principles

  • Neutral Professional Australian English — informed by ICE-AUS corpus research. No stereotyped slang; approachable and direct.
  • RAG-Ready Data Schema — Price and Stock are separated from static product data because they change frequently (the "Frozen in Time" Paradox in AI training).
  • Content Triage — Marketing copy is separated from technical specs to prevent the AI from using promotional language in support contexts.
  • Anti-Greenwashing — Only specific, verifiable claims (e.g., "OEKO-TEX Standard 100 certified") rather than vague terms like "eco-friendly."
  • Phantom Stock Prevention — Stock levels include buffer logic to prevent the AI from confirming availability of items that are nearly depleted.

Technical Implementation

  • Static HTML/CSS/JS site hosted on GitHub Pages
  • Product data served from a JSON file acting as a lightweight database
  • Dynamic product pages with JavaScript rendering from JSON data
  • Chatbot UI (Bilby) with keyword-matching responses — demonstrating the frontend template for a future AI-powered assistant
  • WCAG 2.1 AA accessibility standards considered throughout
  • No backend, no real payment processing, no real user accounts
